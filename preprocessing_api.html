Módulo de Pré-processamento¶

Esta seção detalha os componentes responsáveis pela preparação e transformação dos dados.

Engenharia de Features¶

Módulo de engenharia de features para o pipeline de detecção de fraudes.

Este módulo implementa um pipeline de pré-processamento de dados utilizando classes que seguem o padrão do scikit-learn (BaseEstimator, TransformerMixin). Cada classe representa uma etapa específica da transformação de dados, como agrupamento de categorias, tratamento de países e codificação de variáveis.

A classe principal, FeatureEngineer, orquestra a execução de um pipeline sequencial que aplica todas as transformações, lendo os dados brutos e salvando o resultado processado.

Classes de Transformação:

  • CategoryGrouper: Agrupa categorias de produtos com baixa representatividade em uma única categoria “outros”.

  • CountryGrouper: Agrupa países menos relevantes em uma categoria “Outros”.

  • DocumentFeatureCreator: Cria novas features binárias baseadas na entrega de documentos.

  • ColumnEncoder: Codifica colunas categóricas para valores numéricos usando LabelEncoder.

Orquestrador:

  • FeatureEngineer: Gerencia a leitura, processamento via pipeline e salvamento dos dados.

class fundamentos_engenharia_software.preprocessing.feature_engineering.CategoryGrouper(percentage_cutoff=80, top_n=685)¶

Base: BaseEstimator, TransformerMixin

Agrupa categorias de produtos com baixa representatividade de fraudes.

Este transformador primeiro calcula a porcentagem acumulada de fraudes por categoria de produto para identificar as categorias menos relevantes. Em seguida, no passo de transformação, agrupa essas categorias em um único valor “categorias_outros”.

Parâmetros:

  • percentage_cutoff (int) – O ponto de corte percentual acumulado de fraudes para considerar uma categoria como relevante.

  • top_n (int) – O índice a partir do qual as categorias serão selecionadas para agrupamento. Este é um parâmetro de corte adicional.

Variáveis:

least_frequent_categories (list) – Lista de nomes das categorias de produtos que foram identificadas durante o fit para serem agrupadas. O sufixo _ indica que este atributo é aprendido e armazenado após o fit.

fit(X, y)¶

Aprende quais categorias de produtos agrupar com base na frequência de fraudes.

Calcula a distribuição de fraudes por categoria e armazena uma lista daquelas que contribuem menos para o total de fraudes.

Parâmetros:

  • X (DataFrame) – O DataFrame de features, que deve conter a coluna ‘categoria_produto’.

  • y (Series) – A série do alvo, que deve conter a variável ‘fraude’.

Retorna:

A própria instância do objeto ajustado (fitted).

Tipo de retorno:

CategoryGrouper

transform(X)¶

Aplica o agrupamento de categorias ao DataFrame.

Cria uma nova coluna ‘grupo_categorias’ onde as categorias menos frequentes (aprendidas no fit) são substituídas por ‘categorias_outros’.

Parâmetros:

X (DataFrame) – O DataFrame de entrada para transformar.

Retorna:

O DataFrame com a nova coluna ‘grupo_categorias’.

Tipo de retorno:

DataFrame

class fundamentos_engenharia_software.preprocessing.feature_engineering.ColumnEncoder(*args: Any, **kwargs: Any)¶

Base: BaseEstimator, TransformerMixin

Codifica colunas categóricas (object) em valores numéricos usando LabelEncoder.

Variáveis:

  • encoders (dict) – Dicionário que armazena os encoders ajustados para cada coluna.

  • columns (list) – Lista das colunas que foram identificadas como categóricas e codificadas.

fit(X, _y=None)¶

Ajusta um LabelEncoder para cada coluna do tipo ‘object’.

Parâmetros:

X (DataFrame) – O DataFrame de onde as colunas categóricas serão identificadas e os encoders ajustados.

Retorna:

A própria instância do objeto ajustado (fitted).

Tipo de retorno:

ColumnEncoder

transform(X)¶

Aplica o LabelEncoding às colunas usando os encoders já ajustados.

Parâmetros:

X (DataFrame) – O DataFrame a ser transformado.

Retorna:

Cópia do DataFrame com as colunas categóricas codificadas.

Tipo de retorno:

DataFrame

class fundamentos_engenharia_software.preprocessing.feature_engineering.CountryGrouper(countries_to_keep)¶

Base: BaseEstimator, TransformerMixin

Agrupa países em ‘Outros’, mantendo apenas os especificados.

Cria a coluna ‘paises_agrupados’, que contém o valor original da coluna ‘pais’ se ele estiver na lista de países a manter, ou ‘Outros’ caso contrário.

Parâmetros:

countries_to_keep (List[str]) – Lista de códigos de países que não devem ser agrupados.

fit(_X, _y=None)¶

Fit vazio, sem aprendizado necessário

Tipo de retorno:

CountryGrouper

transform(X)¶

Aplica o agrupamento de países.

Parâmetros:

X (DataFrame) – DataFrame com a coluna ‘pais’.

Retorna:

Cópia do DataFrame com a nova coluna ‘paises_agrupados’.

Tipo de retorno:

DataFrame

class fundamentos_engenharia_software.preprocessing.feature_engineering.DocumentFeatureCreator(*args, **kwargs)¶

Base: BaseEstimator, TransformerMixin

Cria novas features baseadas nas colunas de entrega de documentos.

Adiciona duas novas colunas: 1. ‘entrega_doc_2_nan’: Flag (1/0) que indica se ‘entrega_doc_2’ é nulo. 2. ‘entrega_doc’: Flag que indica se algum dos documentos foi entregue.

fit(_X, _y=None)¶

Fit vazio, sem aprendizado

Tipo de retorno:

DocumentFeatureCreator

transform(X)¶

Aplica a criação das features de documentos.

Parâmetros:

X (DataFrame) – DataFrame com as colunas ‘entrega_doc_1’, ‘entrega_doc_2’, e ‘entrega_doc_3’.

Retorna:

Cópia do DataFrame com as duas novas colunas de features.

Tipo de retorno:

DataFrame

class fundamentos_engenharia_software.preprocessing.feature_engineering.FeatureEngineer(input_path, output_path, artifacts_path)¶

Base: object

Orquestra o pipeline completo de engenharia de features.

Esta classe encapsula todo o processo: - Leitura dos dados brutos de um arquivo de entrada. - Execução de um pipeline de transformações. - Salvamento do DataFrame processado em um arquivo de saída.

Parâmetros:

  • input_path (str) – O caminho para o arquivo de dados brutos (ex: .xlsx).

  • output_path (str) – O caminho onde o arquivo CSV processado será salvo.

Variáveis:

  • data (pd.DataFrame) – DataFrame contendo os dados brutos lidos.

  • processed_data (pd.DataFrame) – DataFrame contendo os dados após a aplicação do pipeline.

  • feature_pipeline (Pipeline) – O objeto de pipeline do scikit-learn contendo todas as etapas de transformação.

read_raw_data()¶

Lê os dados brutos do input_path e os armazena no atributo data.

Tipo de retorno:

None

run_feature_engineer()¶

Executa o fluxo completo de engenharia de features.

Este método orquestra a leitura, a separação de X e y, a aplicação do pipeline de fit_transform, e o salvamento dos resultados.

Tipo de retorno:

None

save_feature_engineer_results()¶

Salva o DataFrame processado (processed_data) no output_path.

Tipo de retorno:

None
